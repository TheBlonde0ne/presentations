Methanol Workflow

From foundation-model sampling to DFT relabeling, MLIP training, and production MD

Project Goal

One workflow, multiple engines

Sample methanol configurations with a foundation MACE model, refine a small subset with DFT, correct the data with D3, train and compare MLIPs, and then use the best model for production MD.

  • Build a reproducible pipeline instead of ad hoc scripts.
  • Track dependencies between sampling, labeling, training, and evaluation.
  • Compare the final models on structure and dynamics, especially RDFs and diffusion.

Why Use a Workflow Manager?

More than automation

A workflow manager is useful because the project combines MD, DFT, and MLIP training in one chain. It keeps the logic organized, handles dependencies between stages, and makes reruns predictable.

  • Reproducibility: every step is defined and rerunnable.
  • Scalability: the same pipeline can be expanded to more data or more temperatures.
  • Traceability: outputs from each stage stay linked to the node that produced them.

The central idea is simple: use the fast foundation model to explore phase space, then spend the expensive DFT budget only where it matters.

Sampling With MACE

Generate methanol configurations and sample MD frames with the MACE foundation model

  • Start from initial configurations generated with Packmol.
  • Use the pretrained MACE foundation model for fast MD sampling.
  • Inspect the resulting trajectory with zntrack.from_rev() and the frames list.

Why this matters

The foundation model provides cheap, broad sampling of methanol configurations before any expensive DFT labeling is done.

Stage 2: DFT Relabeling

  • Run a convergence test on a small number of sampled frames.
  • Vary the CP2K MGRID cutoff until force and energy changes are below the target threshold.
  • Relabel the trajectory with the optimal DFT settings.
  • Add D3 correction outside CP2K when possible, because it depends only on the atoms and is cheaper to evaluate separately.

What the MGRID test gives you

A practical accuracy-cost tradeoff: enough grid resolution to converge forces and energies, but not so much that DFT becomes unnecessarily expensive.

Stage 3: Train MLIPs

Hyperparameter exploration

Train several models while varying the number of training configurations, radial cutoff rmax, basis size, and network depth or width.

  • More data usually improves accuracy, but increases training cost.
  • Larger cutoffs help with longer-range interactions, but are more expensive.
  • More basis functions and larger networks can fit more complex behavior, but raise overfitting risk.
  • Train ensembles to compare uncertainty through prediction spread.

Raw DFT vs D3-Corrected Data

Raw DFT

Use the labeled CP2K data directly to train one model and keep the full electronic-structure reference as-is.

D3-corrected

Train a second model on DFT plus the separately evaluated dispersion contribution to better capture intermolecular attraction.

Comparing both models helps isolate how much of the final performance comes from the base DFT functional versus the dispersion correction.

Stage 4: Production MD and Evaluation

  • Select the best-performing MLIP from the hyperparameter study.
  • Run longer production MD simulations with the chosen model.
  • Evaluate structural observables such as RDFs and dynamic quantities such as diffusion.
  • Compare uncertainty and stability across single models and ensembles.

Final question

Does the trained model preserve methanol structure and transport well enough to be trusted for production-scale sampling?

Takeaways

Fast sampling, selective DFT labeling, dispersion correction, and MLIP screening form a single iterative loop

Summary

  • MACE provides cheap trajectory sampling for methanol.
  • MGRID convergence identifies a practical DFT setting for labeling.
  • D3 correction improves the physical reference data used for training.
  • Hyperparameter search and ensembles improve robustness and uncertainty estimates.
  • The best MLIP is validated on production MD with RDF and diffusion analysis.
